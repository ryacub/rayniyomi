# Reader TalkBack Parity Checklist

Run this checklist on a stable benchmark build before promoting baseline updates.

- [ ] Reader page announcement is spoken when entering a chapter.
- [ ] Overlay controls are reachable in deterministic order.
- [ ] Error surface heading is discoverable as a heading.
- [ ] Retry control is announced and actionable.
- [ ] Open-in-web fallback control (when shown) is announced and actionable.
- [ ] Focus returns to reader content after dismissing overlays/dialogs.
